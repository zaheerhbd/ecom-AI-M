eCOM-AI RAG Next Steps

Project snapshot generated on April 19, 2026

What Is Already Working

  • Azure AI Search service is created and reachable.
  • Product index exists and is aligned closely enough to accept product data.
  • Products from PostgreSQL can be synced into Azure AI Search.
  • Documents are visible in Azure Search Explorer. Done
  • AI document generation exists for products and delivery methods.
  • Embedding generation exists with OpenAI and a local fallback.
  • Retriever logic exists and can rank relevant documents.
  • `POST /ai/chat` exists, but currently returns a formatted answer rather than a true LLM-generated grounded answer.

Highest-Priority Next Steps

  • Add a real generation step in `AiChatService` so retrieved context is sent to an LLM and the answer is generated from that context.
  • Build a grounding prompt that tells the model to answer only from retrieved product and delivery data.
  • Return source citations clearly in the AI response so the frontend can show where the answer came from.
  • Add confidence handling for weak retrieval cases so the assistant can say it does not know instead of bluffing.

Search And Retrieval Tasks

  • Decide whether Azure AI Search will be used only for product search or also for RAG retrieval.
  • If Azure AI Search will be part of RAG, choose between: keyword search only, hybrid search, or vector search with embeddings.
  • Normalize the index schema so field types are intentional and documented.
  • Consider improving `createdAt` to a date field later if sorting/filtering by date becomes important.

Operational Tasks

  • Add automatic sync for product create, update, and delete events.
  • Add a rebuild or reindex command for recovery scenarios.
  • Add structured logging around AI retrieval and answer generation.
  • Add tests for document shaping, retrieval behavior, and Azure sync mapping.

Suggested Implementation Order

  • Step 1: Make `/ai/chat` a true RAG endpoint using the current retriever.
  • Step 2: Improve answer quality with better prompts and citation formatting.
  • Step 3: Decide whether to keep in-memory retrieval or move chat retrieval into Azure AI Search.
  • Step 4: Add automatic sync hooks for product changes.
  • Step 5: Connect the frontend to the improved AI response flow.
Recommended immediate move: keep the current retriever for now and add the missing LLM generation step first. That is the fastest path from "retrieval exists" to "real RAG works."
