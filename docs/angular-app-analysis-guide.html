Angular Application Analysis Guide

Easy-to-read project walkthrough for an experienced .NET developer learning Angular deeply. This guide explains the business flow, technical flow, routing, services, forms, RxJS, security, performance, and interview-ready talking points.

Project Type: Angular 11 + .NET 5 E-Commerce Frontend Pattern: NgModules + Lazy Loading State Style: Shared Services + RxJS Subjects Payments: Stripe

How To Use This Guide

  1. Read the overview first to understand what the app does.
  2. Use the structure and routing sections to build a mental map.
  3. Study components and services together to connect screen to API.
  4. Use the interview sections to practice confident explanations.
Think of this app like a .NET MVC application split into Angular feature areas. Components are like UI controllers + views combined, and services are like client-side application services.

1. Application Overview

This is an AI-enhanced e-commerce application. A user can browse products, search and filter them, add items to a basket, log in, complete checkout with Stripe, and review past orders. There is also an AI-Lab page where the user can ask product-related questions in natural language.

3. Routing

Routing is handled using NgModule-based Angular Router. The root route file lazy-loads most feature modules. This reduces first-load size and improves startup performance.

6. State Management

This app uses a service-based state management approach instead of NgRx. That is a common and reasonable choice for a medium-sized app.

State Type How It Is Managed Why It Works Here
User auth state ReplaySubject<IUser> in AccountService Late subscribers still get the latest user
Basket state BehaviorSubject<IBasket> in BasketService Components can get current basket immediately
Basket totals Separate BehaviorSubject Keeps summary data ready for basket and checkout
Shop filter state Plain service fields in ShopService Simple and enough for the catalog page
Persisted state localStorage for token and basket id Survives refresh
Simple analogy: this is like keeping lightweight app state in singleton services instead of introducing a full Redux-style store.

8. RxJS

RxJS Feature Where Used Why It Is Used Interview Question
BehaviorSubject Basket service Holds latest basket and totals Why choose BehaviorSubject for shared state?
ReplaySubject(1) Account service Replays latest user to new subscribers ReplaySubject vs BehaviorSubject?
map Service response transforms Shapes data before component uses it What does map do in RxJS?
switchMap Register email validator Cancels old validation requests when typing continues Why is switchMap good for search or validation?
catchError Error interceptor Centralized HTTP error handling Where should HTTP error handling live?
async pipe Navbar, basket, totals, breadcrumbs Auto-subscribes and cleans up in templates Why prefer async pipe in templates?
Important observation: the app does not use takeUntil or OnPush, and there are several manual subscriptions. That is not automatically wrong for one-time HTTP requests, but it is a good improvement topic in interviews.

9. Authentication and Authorization

Login Form
→

→
Receive token
→

→

  • Authentication is token-based.
  • The token is saved in localStorage.
  • JwtInterceptor adds the bearer token to outgoing requests.
  • AuthGuard protects checkout and orders routes.
  • Login supports returning the user to the originally requested page using returnUrl.
Interview-friendly explanation “We used route guards for client-side protection, but real authorization is enforced on the API. The Angular app stores the token, rehydrates user state on startup, and uses an interceptor to attach the bearer token automatically.”

11. Performance

Good Choices Already Present

  • Lazy-loaded feature modules
  • Shop data caching
  • Async pipe in several shared screens
  • Resolver for product details

Improvements To Mention

  • Add OnPush where suitable
  • Add trackBy for large lists
  • Reduce repeated async evaluations in templates
  • Replace deprecated toPromise()
  • Use debounced search
Senior-level explanation “The app already uses lazy loading and some useful caching, but I would still improve change detection strategy, list rendering stability, and observable handling to make it scale better.”

12. Security

  • Protected backend endpoints are a strong point.
  • No obvious unsafe innerHTML usage was found.
  • Stripe secret remains on the backend, which is correct.
  • Client stores token in localStorage, which is convenient but can be a risk if XSS exists.
  • Route guards help UX but are not the true security boundary. The API is.
Good interview sentence: “Client-side protection improves navigation behavior, but actual authorization must always be enforced by the server.”

13. Interview Explanation Pack

Tell me about this Angular project. “It is an Angular 11 e-commerce frontend integrated with a .NET 5 backend. The application supports product browsing, filtering, basket management, authentication, multi-step checkout with Stripe, order tracking, and an AI-assisted shopping experience.”
How does routing work in your application? “The app uses a root router with lazy-loaded feature modules. Product and order details use route parameters, protected routes use an auth guard, and product details also use a resolver so data is ready before the screen renders.”
How do components communicate? “Parent-child communication is handled through Input and Output bindings, and cross-feature shared state like the current user or basket is handled through singleton services using RxJS subjects.”
How do you call APIs? “Components delegate network work to feature services built on HttpClient. Interceptors handle JWT attachment, loading indicators, and centralized HTTP error behavior.”
How do you manage state? “This app uses service-based state management rather than NgRx. Basket and user state are stored in RxJS subjects, while local component state handles view-specific concerns.”
How do you improve performance? “The project already uses lazy loading and some response caching. My next steps would be OnPush change detection, trackBy for lists, and more reactive search handling.”
What would you improve? “I would strengthen client logging, modernize some async handling, improve change detection strategy, and tighten some template and observable patterns for maintainability and scale.”

15. Learning Plan

Best Order To Study

  1. Read AppModule, AppRoutingModule, and AppComponent.
  2. Study the shop flow: list page, detail page, shop service, models.
  3. Study basket state: basket component, basket service, shared basket summary component.
  4. Study authentication: login, register, account service, auth guard, JWT interceptor.
  5. Study checkout carefully: parent form, child steps, payment flow, order creation.
  6. Study error interceptor and loading interceptor as cross-cutting concerns.
  7. Finish with orders and AI-Lab as advanced feature understanding.

Angular Concepts You Must Learn From This App

  • NgModules and feature modules
  • Lazy loading
  • Router parameters, query parameters, guards, resolvers
  • Reactive forms and async validators
  • Custom form controls with ControlValueAccessor
  • HttpClient and interceptors
  • RxJS subjects and async pipe
  • Service-based state management
  • Integration with third-party libraries like Stripe
If you learn those topics from this app, you will be able to explain Angular architecture much more confidently in interviews.
